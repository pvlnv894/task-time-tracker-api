# Task Time Tracker API
REST-API для учета времени, которое сотрудники тратят на задачи. 

---

## Функционал
- **Управление задачами**: создание задач, получение информации по ID, смена статуса (NEW, IN_PROGRESS, DONE).
- **Учет времени**: фиксация временных отрезков работы сотрудника над конкретной задачей.
- **Отчетность**: получение истории трудозатрат сотрудника за выбранный период времени.

## Реализованный функционал

### Эндпойнты (Tasks):
- `POST /tasks` - Создание новой задачи.
- `GET /tasks/{id}` - Получение информации о задаче.
- `PATCH /tasks/{id}/status` - Изменение статуса задачи (NEW, IN_PROGRESS, DONE).

### Эндпойнты (Time Records):
- `POST /time-records` - Фиксация временного отрезка работы над задачей.
- `GET /time-records` - Получение статистики трудозатрат сотрудника за период (фильтры по `startTime`, `endTime`, `employeeId`).

## Тестирование и валидация
### Unit-тесты 
Покрытие бизнес-логики сервисов с использованием Mockito.

### Интеграционные тесты
Проверка DAO-слоя (мапперов MyBatis) с помощью **Testcontainers**

### Bean Validation
Входные DTO валидируются на уровне контроллеров (проверки на `null` и пустые строки).

### Rest Controller Advice
Реализована глобальная обработка исключений через `@RestControllerAdvice` для возврата понятных ошибок клиенту.

## Использование ИИ

В процессе выполнения задания в качестве ИИ-ассистента использовался **Google Gemini**. 

### Артефакты взаимодействия:
1. **Проектирование и реализация DAO-слоя**: Помощь в разработке XML-мапперов MyBatis.
2. **Интеграция документации OpenAPI в код**: Помощь с внедрением аннотаций SpringDoc / Swagger (@Operation, @ApiResponse) в контроллеры.
3. **Разработка стратегии тестирования**: Использование ArgumentCaptor (Mockito) для перехвата объектов внутри void-методов.
4. **Проектирование тестов**: Помощь в настройке конфигурации Testcontainers для работы с PostgreSQL и MyBatis.

*(Скриншоты ключевых этапов взаимодействия с ИИ находятся в папке `/docs/ai-artifacts`)*.
